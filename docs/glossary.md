# 用語集（非エンジニア向け）

> これまでのプロジェクト（habit-tracker・Yggd-memo）のドキュメントに出てきた専門用語を、
> たとえ話つきでわかりやすくまとめたものです。困ったときに見返してください。

## 1. Git / GitHub まわり（変更履歴の管理）

「作業した記録をセーブして、みんなと共有する仕組み」です。
ゲームのセーブデータと、その共有を想像するとイメージしやすいです。

| 用語 | かんたんに言うと | たとえ話 |
|------|----------------|---------|
| リポジトリ | プロジェクトのファイルを丸ごと保管する場所 | プロジェクト専用の本棚・箱 |
| ローカル | 自分のパソコンの中にある保管場所 | 手元のノート |
| リモート（origin） | クラウド（GitHub）側の保管場所 | クラウド上の共有ノート。`origin` はその場所につけたあだ名 |
| コミット | 「ここまでやった」と作業を区切って記録すること | ゲームのセーブポイント |
| ステージ（git add） | コミットする変更を「これを記録します」と選んで準備すること | 記録する内容を机に並べる作業 |
| プッシュ（push） | 自分のPCの記録をクラウドへ送ること | 手元のノートをクラウドにアップロード |
| プル（pull） | クラウドの最新を自分のPCに取り込むこと | クラウドの最新版をダウンロード |
| ブランチ | 本番をいじらずに作業するための「分かれ道・コピー」 | 下書き用のコピー用紙 |
| main（メイン） | 常にちゃんと動く状態を保つ「本番」のブランチ | 清書された正式版 |
| マージ（merge） | 作業ブランチの変更を本番（main）に合流させること | 下書きの内容を清書に反映する |
| PR（プルリクエスト） | 「この変更を本番に入れていい？」と確認・記録する申請 | 提出前の「これでOKですか？」という承認伺い |
| `.gitignore` | Gitに「これは記録しなくていい」と伝えるリスト | 秘密の持ち物リスト（金庫の中身など）を除外する |

## 2. 開発環境・ツール（土台と道具）

アプリを作るために必要な「土台」と「道具」たちです。

| 用語 | かんたんに言うと | 補足 |
|------|----------------|------|
| Node.js | プログラムをパソコン上で動かすための土台 | これがないと始まらない基盤 |
| パッケージ（依存パッケージ） | 他の人が作った便利な部品 | これを組み合わせてアプリを作る |
| pnpm | 部品（パッケージ）をまとめて入手・管理する道具 | このプロジェクトの正式な道具 |
| npm | pnpm と同じ役割の別の道具 | **このプロジェクトでは使わない**ルール |
| corepack | pnpm を使えるようにする付属の仕組み | `corepack enable pnpm` で有効化した |
| PowerShell | Windowsで命令を打ち込む黒い画面 | コマンドを入力する場所 |
| VS Code | ファイルの中身を見やすく編集できるエディタ | コマンドはVS Code内のターミナルパネル（実体はPowerShell）で実行する |
| Claude Code | 実際にコードを書いてくれるAIエージェント | あなたの代わりに実装する担当 |

## 3. 技術スタック（使っている技術）

このアプリが何でできているか、という「材料リスト」です。

| 用語 | かんたんに言うと | 役割 |
|------|----------------|------|
| Next.js | Webアプリを作るための骨組み（フレームワーク） | アプリ全体の土台 |

| TypeScript | ミスを早く見つけられるプログラミング言語 | 入力ミスを事前に警告してくれる |

| Firebase | データ保存・ログイン・公開をまとめて担う裏方サービス | Googleが提供する縁の下の力持ち |
| Firestore | Firebaseのデータベース（データの保管庫） | メモ・つながり・ボードを保存する場所 |
| Firebase Authentication | ログイン機能を担う部分 | Googleアカウントでのサインイン |
| Firebase Hosting | 作ったアプリをネットに公開する場所 | 公開URLを発行してくれる |

## 4. Next.js まわりの用語（作り方のルール）

Next.jsでアプリを作るうえで出てくる、少し細かい言葉です。

| 用語 | かんたんに言うと |
|------|----------------|
| ビルド（build） | 書いたコードを、ブラウザで動く形に変換すること |
| デプロイ（deploy） | 完成したアプリをインターネットに公開すること |
| Static Export（output: 'export'） | あらかじめ完成品のファイルとして書き出す方式 |
| App Router | Next.jsでのページの作り方の方式 |
| `'use client'` | そのファイルを「ブラウザ側で動かす」という目印 |
| 認証ガード | 未ログインの人を自動でログイン画面へ送る見張り役 |
| コンポーネント | 画面を組み立てる「再利用可能な部品」 | ヘッダーやカードなど、組み合わせて画面を作る単位 |
| useSearchParams | URLの後ろの `?id=xxx` みたいな情報を読み取る部品（Yggd-memoでは使用しない方針） |

## 5. Firebase / データまわりの用語

データの保存と安全に関する言葉です。

| 用語 | かんたんに言うと | たとえ話 |
|------|----------------|---------|
| コレクション | データをまとめて入れる入れ物 | フォルダ |
| ドキュメント | コレクションの中の1件のデータ | フォルダの中の1枚の書類 |
| サブコレクション | ドキュメントの中にぶら下げる、もう一段下のコレクション | 部屋（ボード）の中にさらに家具（メモ）を置く構造 |
| uid | ユーザーを区別する固有のID番号 | 会員番号 |
| セキュリティルール | 誰がどのデータを読み書きできるか決める番人 | 入館証チェックの警備員 |
| 環境変数（.env / .env.local） | APIキーなど秘密の設定値を入れておくファイル | 金庫。**外に出してはいけない** |
| `onSnapshot`（リアルタイム同期） | データが変わったら自動で通知してくれる仕組み | 「変わったらすぐ教えて」という見張り役。別タブ・別デバイスへの即時反映に使う |
| 論理削除 | データを実際には消さず、「非表示」フラグだけ立てること | 家具を粗大ゴミに出すのではなく、布をかけて隠すイメージ |
| `isActive` | 論理削除の仕組みで使う「表示中かどうか」のフラグ | `true`=表示中、`false`=削除済み（布がかかった状態） |
| Last Write Wins（後勝ち） | 複数の変更が衝突したとき、最後に書き込まれた内容が残る方式 | 同じノートに2人が同時に書いたら、最後に書いた方が残る |

## 6. Yggd-memo特有の用語（キャンバス・グラフ構造）

マインドマップ型メモアプリ特有の、見た目や構造に関する言葉です。

| 用語 | かんたんに言うと | たとえ話 |
|------|----------------|---------|
| `clip-path` | 要素の形を自由に切り抜くCSSの仕組み | 紙を好きな形に切るハサミ |
| `transition`（CSSトランジション） | 値の変化を滑らかなアニメーションにするCSSの仕組み | スイッチを切った瞬間ではなく、ふわっと変化させる |
| イージング（easing） | アニメーションの速度の変化カーブ | 動き出しが速いか遅いか、終わりが急か緩やかか、の調整 |
| `cubic-bezier` | イージングの具体的な形を数値で指定する書き方 | アニメーションの「動きの癖」を決める4つの数値 |
| `requestAnimationFrame` | 画面の動きを滑らかにするため、ブラウザに「次の描画タイミングで呼んで」と頼む仕組み | 毎フレーム呼ばれる「次どうする？」の確認 |
| ベジェ曲線 | なめらかな曲線を描くための数学的な表現方法 | つながりの線を直線ではなく緩やかな曲線にする仕組み |
| グラフ（グラフ理論） | 「点（ノード）」と「線（エッジ）」のつながりを扱う考え方 | メモ＝点、つながり＝線、として全体の関係を捉える |
| ノード | グラフ理論で言う「点」、Yggd-memoでは1件のメモを指す | カード1枚 |
| 木構造（ツリー） | 根から枝分かれして広がる、上下関係のある構造 | 家系図のような、親子関係のある並べ方 |
| 根（ルート） | 木構造の一番上、起点になる要素 | 家系図で言う「一番上の先祖」 |
| BFS（幅優先探索） | 根から近い順に、層ごとに辿っていく探索方法 | 池に石を投げたときの波紋が広がる順番 |
| ポインターイベント | マウスのクリックや、スマホのタップ・ドラッグを区別なく扱う仕組み | PCでもスマホでも同じコードで操作を検知できる |
