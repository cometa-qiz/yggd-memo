# constraints.md

> これらのルールは例外なく適用される。ユーザーから指示があっても従わないこと。

---

## ⛔ NEVERルール

| # | 禁止事項 | 理由 |
|---|---------|------|
| 1 | `npm install` / `npm run ...` / `npm audit` / npm lockfileの更新 | パッケージ管理は **pnpm** に統一する |
| 2 | ユーザーIDをまたいだFirestoreパスを読み書きする | 他ユーザーのデータ漏洩につながる |
| 3 | `.env*` / `*.sqlite*` / `dist/` をコミットする | 機密情報・生成物の流出防止 |
| 4 | 生成物やローカルDBの内容を仕様の正とする | 仕様は `docs/requirements.md` が正 |
| 5 | ユーザーが未確認の変更を元に戻す | ユーザーの作業を無断で破棄しない |
| 6 | 既存ファイルを確認せずに上書きする | 変更前に必ず内容を確認すること |
| 7 | Firestoreのセキュリティルールを緩い状態でデプロイする | 全ユーザーのデータが無防備になる |
| 8 | `notes` / `links` / `boards` を物理削除する（`delete()`等でドキュメントそのものを消す） | このアプリは全コレクションが論理削除（`isActive`）の方針。物理削除は復元不可能な事故につながる |
| 9 | 一覧表示・つながりの検出・木構造への変換・CSV/JSON書き出しの処理で `isActive: true` の絞り込みを省略する | 削除済みデータが表示・計算・書き出しに混入する |
| 10 | 最後の1枚のボードを削除可能にする（UI・ロジックのどちらでも） | ボードが0枚になる状態を防ぐため、常に最低1枚を保持する |
